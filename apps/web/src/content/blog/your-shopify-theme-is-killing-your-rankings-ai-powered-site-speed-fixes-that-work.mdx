Most Shopify store owners choose themes for aesthetics, not performance. The result:
- **Average Shopify store PageSpeed score**: 35-55 on mobile
- **Revenue impact**: Every 100ms of added load time reduces conversion by 1-2%
- **SEO impact**: Core Web Vitals are a confirmed Google ranking factor
- **User impact**: 53% of mobile visitors leave if a page takes more than 3 seconds

## The Top 5 Theme Performance Killers

### 1. Unoptimised Hero Sliders
Auto-playing image sliders load multiple full-size images on page load:
- **Fix**: Replace sliders with a single hero image. If you must use a slider, lazy load all but the first slide.
- **Impact**: LCP improvement of 1-3 seconds

### 2. Too Many App Scripts
Each Shopify app injects JavaScript. 10 apps = 10 extra script loads:
- **Fix**: Audit all apps. Remove unused ones. Defer remaining app scripts.
- **Impact**: INP improvement of 100-300ms

### 3. Oversized Product Images
Shopify themes often request full-resolution images regardless of display size:
- **Fix**: Use Shopify's image API with proper width parameters. Serve WebP format.
- **Impact**: LCP improvement of 0.5-2 seconds

### 4. Render-Blocking CSS
Themes loading entire CSS files synchronously before any content displays:
- **Fix**: Inline critical CSS, defer non-critical CSS.
- **Impact**: FCP improvement of 0.5-1.5 seconds

### 5. Missing Image Dimensions
Images without explicit width and height cause layout shift as they load:
- **Fix**: Add width and height attributes to all image tags.
- **Impact**: CLS reduction from 0.3+ to under 0.1

## When to Fix vs When to Replace Your Theme

**Fix your current theme if:**
- You've customised it extensively
- Performance issues are specific and fixable
- The theme structure is fundamentally sound

**Replace your theme if:**
- PageSpeed score is consistently below 30
- The theme hasn't been updated in 2+ years
- Multiple structural performance issues exist
- The theme doesn't support modern Shopify features (sections, metafields)
