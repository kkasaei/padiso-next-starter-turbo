## Multi-Store Content Architecture

For Shopify Plus brands with multiple stores, establish a content hierarchy:

### Global Content Layer
- Brand voice guidelines and tone documentation
- Product naming conventions and attribute standards
- Universal SEO rules (schema types, heading structure, keyword strategy)
- Compliance requirements that apply across all markets

### Store-Specific Content Layer
- Locale-specific product descriptions
- Market-tailored collection page content
- Region-specific blog topics and editorial calendar
- Local SEO elements (GMB integration, local keywords)

### Product-Level Content Layer
- Individual product descriptions optimized per locale
- Product-specific FAQ content
- Schema markup tailored to each market's rich result opportunities
- Meta tags optimized for each locale's search patterns

Claude 4.6 Opus manages all three layers simultaneously, ensuring global consistency with local relevance.

## Workflow for Multi-Store Content Generation

### Phase 1: Global Standards Setup
1. Feed Claude 4.6 Opus your complete brand guidelines
2. Define SEO standards for each content type (product, collection, blog, meta)
3. Establish compliance rules for each target market
4. Create the master prompt library

### Phase 2: Per-Store Content Generation
For each store/locale:
1. Import product catalog data
2. Research market-specific keywords
3. Generate all content types using store-specific prompts
4. Validate against brand and compliance standards

### Phase 3: Cross-Store Optimization
1. Ensure no content duplication between stores
2. Verify hreflang implementation across all locales
3. Check internal linking structures per store
4. Validate schema markup across all storefronts
