## What Is Claude Code?

Claude Code is Anthropic's AI-powered coding assistant that runs in your terminal. It can:

- **Understand your entire Shopify theme** by reading all your template files
- **Generate Liquid code** from natural language descriptions
- **Modify existing files** with precise, targeted edits
- **Test and validate** that generated code follows Shopify best practices

For SEO-focused Shopify work, it's a game-changer. You describe the SEO feature you want, and Claude Code builds it.

## SEO Section 1: Dynamic FAQ Schema Section

Every product and collection page benefits from FAQ content with proper schema markup. Here's how to build it with Claude Code:

**What to tell Claude Code:**
"Create a Shopify section called 'seo-faq' that lets me add FAQ questions and answers through the theme customizer. It should output FAQ content visually on the page and also inject FAQPage JSON-LD schema in the page head."

**What you get:**
- A reusable section file with block settings for unlimited Q&A pairs
- Proper JSON-LD FAQPage schema automatically generated from the section blocks
- Clean, accessible HTML with expand/collapse functionality
- Mobile-responsive styling

## SEO Section 2: Collection SEO Content Block

Collection pages need editorial content for SEO but most themes don't include a content section:

**What to tell Claude Code:**
"Create a section called 'collection-seo-content' that appears below the product grid on collection pages. It should have a rich text editor field, support H2 headings, and include a metafield option for the SEO text so it can be managed per collection."

**What you get:**
- A section that reads from collection metafields or manual input
- Proper heading hierarchy (H2, H3) for SEO structure
- Conditional rendering—only shows if content exists
- Read more/less toggle for longer content

## SEO Section 3: Breadcrumb Navigation

Proper breadcrumbs improve both UX and SEO through BreadcrumbList schema:

**What to tell Claude Code:**
"Create a breadcrumb snippet that works on product and collection pages. It should show Home > Collection > Product, with proper BreadcrumbList JSON-LD schema. Handle products in multiple collections by using the collection the customer navigated from."

**What you get:**
- Context-aware breadcrumbs that reflect the user's navigation path
- BreadcrumbList schema markup matching the visible breadcrumbs
- Styling that matches your theme's design system
- Proper fallback when no collection context is available

## SEO Section 4: Product Structured Data Enhancement

Shopify's default product schema is minimal. Enhance it:

**What to tell Claude Code:**
"Enhance the product JSON-LD schema on my product template to include: material from the product metafield 'custom.material', color from the selected variant, AggregateRating from the Shopify product reviews, brand from the vendor field, and availability based on inventory."

**What you get:**
- Comprehensive Product schema that pulls from metafields, variants, and reviews
- Proper handling of product variants as individual Offer objects
- AggregateRating schema from your review system
- Dynamic availability that updates with inventory

## SEO Section 5: Internal Linking Widget

Contextual internal links boost SEO and keep users on your site:

**What to tell Claude Code:**
"Create a section called 'related-content' that displays 3-4 related blog posts on product pages and 3-4 related products on blog posts. Use Shopify's product tags and blog post tags to find matches."

**What you get:**
- Tag-based content matching between products and blog posts
- Responsive card layout for related content
- Proper internal linking with descriptive anchor text
- Configurable through the theme customizer

## The Non-Developer Workflow

Here's the complete process for building these sections:

1. **Set up your local theme** using Shopify CLI (`shopify theme pull`)
2. **Open the theme folder** in your terminal
3. **Run Claude Code** and give it context about your theme
4. **Describe the section you want** in plain English
5. **Review the generated code** (Claude explains what each part does)
6. **Preview on your development store** (`shopify theme dev`)
7. **Push to your live theme** once tested (`shopify theme push`)

## Tips for Working with Claude Code

- **Be specific about your requirements**—"add FAQ schema" is vague; "create a section with FAQ blocks that outputs FAQPage JSON-LD" is precise
- **Provide context about your theme**—tell Claude which theme you're using (Dawn, Debut, custom) so it matches conventions
- **Test one section at a time**—don't try to build everything at once
- **Ask Claude to explain the code**—even if you don't code, understanding what's happening helps you troubleshoot

## Quality Assurance Without Coding Knowledge

After generating each section:

- **Visual check**—does the section look right in your theme preview?
- **Schema validation**—paste the page URL into Google's Rich Results Test
- **Mobile check**—view the section on your phone
- **Performance check**—run PageSpeed Insights before and after adding the section

## Common Issues and How Claude Code Fixes Them

- **"The section doesn't appear on my page"**—Claude can check your template file to ensure the section is properly included
- **"Schema validation shows errors"**—Claude can debug JSON-LD syntax and fix structural issues
- **"The styling looks wrong"**—Claude can adjust CSS to match your theme's design patterns
- **"It works on desktop but not mobile"**—Claude can add responsive styles and fix mobile layout issues

## FAQs

- **Is Claude Code free?**
Claude Code requires an Anthropic API subscription. Costs are based on usage—building these sections typically costs a few dollars.

- **Can I break my store using Claude Code?**
Always work on a development theme, not your live store. Push to live only after thorough testing.

- **What if I need changes later?**
Run Claude Code again and describe the changes. It remembers the context of files it's already seen.

- **Is this better than hiring a Shopify developer?**
For standard SEO sections, yes—it's faster and cheaper. For complex custom features, a developer is still the better choice.
